# 📚 RAG（检索增强生成）面试题

> RAG 系统、向量检索、混合搜索、Graph RAG 等面试核心内容。

---

## 核心概念

**Q1：什么是 RAG？为什么重要？**
> 检索增强生成：先从知识库检索相关文档，再输入 LLM 生成回答。
> 重要性：解决 LLM 知识过期、幻觉、私有数据不可用等问题。

**Q2：RAG 系统的基本架构？**
> ```
> 用户查询 → 查询处理 → 向量检索 → 重排序 → 上下文组装 → LLM 生成 → 答案
> ```
> 关键组件：文档解析 → 分块 → Embedding → 向量数据库 → 检索 → 生成

**Q3：RAG 的关键组件有哪些？**
> - **文档加载器**：PDF、网页、数据库
> - **文本分块器**：将文档分割为适合检索的片段
> - **Embedding 模型**：将文本转为向量
> - **向量数据库**：存储和检索向量
> - **检索策略**：相似度搜索、混合搜索
> - **LLM**：基于检索结果生成答案

**Q4：什么是 Agentic RAG？**
> 将 Agent 能力融入 RAG：Agent 自主决定是否需要检索、检索什么、如何组合多个检索结果。
> 比 naive RAG 更智能，能处理多跳推理、自适应检索。

## 分块策略

**Q5：什么是分块策略？如何选择分块大小？**
> 将长文档分割为适合 Embedding 和检索的小片段。
> 策略：
> - **固定大小**：按 token/字符数切分 + 重叠
> - **递归分块**：按层级结构（标题→段落→句子）递归切分
> - **语义分块**：基于语义相似度边界切分
> 选择：需平衡检索精度（小块）和上下文完整性（大块）

**Q6：固定分块 vs 语义分块 vs 递归分块？**
> | 策略 | 优点 | 缺点 |
> |------|------|------|
> | 固定大小 | 简单可控 | 可能截断语义 |
> | 语义分块 | 保持语义完整 | 需要额外模型，速度慢 |
> | 递归分块 | 兼顾结构和语义 | 实现复杂 |

**Q7：什么是 Parent-Child 分块？**
> 将文档分为小块（child）用于精准检索，检索到后返回其父块（更大上下文）给 LLM。
> 兼顾检索精度和上下文完整性。

## Embedding 与检索

**Q8：什么是 Embedding 模型？如何将文本转为向量？**
> 将文本编码为高维向量（如 768/1536 维），语义相似的文本向量距离近。
> 主流模型：OpenAI ada-002、BGE、E5、Cohere

**Q9：什么是混合搜索？为什么比纯向量搜索更好？**
> 结合**向量搜索**（语义相似）+**关键词搜索**（BM25，精确匹配）。
> 优点：既能找到语义相关的内容，又能匹配专有名词、缩写等。

**Q10：什么是重排序（Re-ranking）？**
> 检索后对结果重新排序，使用更精确的交叉编码器模型打分。
> 两阶段检索：向量数据库粗筛 → Re-ranker 精排。

**Q11：如何处理多文档和多跳问题？**
> - 多文档：为每个文档添加元数据，检索后按来源分组
> - 多跳：迭代检索，将每步结果作为下一步查询
> - 查询分解：将复杂问题拆解为子问题分别检索

**Q12：RAG 中的 "Lost in the Middle" 问题？**
> 检索结果过多时，LLM 对中间位置的上下文关注度最低。
> 解决：限制上下文数量、重排序将重要内容放首尾。

## 高级 RAG

**Q13：什么是 Self-RAG？**
> 模型自己决定是否需要检索、检索结果是否相关、是否生成引用。
> 通过特殊 token（[Retrieve]、[IsRel]、[IsSup]）实现自我反思。

**Q14：什么是 Graph RAG？**
> 将知识组织为图结构（实体+关系），结合图遍历和向量检索。
> 适用：复杂关系推理、多跳问题、实体关系密集的场景。

**Q15：如何处理结构化数据（表格、SQL）？**
> - 表格：Text-to-SQL + 自然语言查询
> - 数据库：NL2SQL + schema 链接
> - 混合方案：结构化查询 + 非结构化 RAG 结合

**Q16：什么是查询变换？**
> - **HyDE**：让 LLM 先生成假设答案，用答案向量检索
> - **查询分解**：将复杂查询拆解为子查询
> - **Step-back Prompting**：先问更抽象的上层问题

## 评估与优化

**Q17：如何评估 RAG 系统？**
> - **Faithfulness**（忠实度）：回答是否基于检索到的上下文
> - **Relevance**（相关性）：检索到的内容与问题的相关度
> - **Context Precision/Recall**：上下文精确率和召回率
> - 工具：RAGAS、TruLens

**Q18：如何在生产中优化 RAG 延迟？**
> - Embedding 模型量化（INT8）
> - 向量索引优化（HNSW、IVF）
> - 缓存热门查询结果
> - 流式生成
> - 异步检索

**Q19：RAG vs 微调 vs Prompt Engineering？如何选择？**
> | 方法 | 适用场景 | 成本 |
> |------|---------|------|
> | Prompt Engineering | 快速原型、通用任务 | 最低 |
> | RAG | 知识密集、需要引用、知识更新频繁 | 中等 |
> | 微调 | 特定格式/风格、领域适应 | 最高 |

**Q20：如何实现引用和来源标注？**
> - 在 Prompt 中要求标注来源段落编号
> - 使用结构化输出（包含 citation 字段）
> - 检索结果附带元数据（来源、页码）

## 🔧 场景排障题

| 场景 | 思路 |
|------|------|
| 有正确上下文但仍幻觉 | Prompt 强制"仅基于上下文回答" + 忠实度检查 + 生成后验证 |
| 分块重叠导致冗余结果 | 减少重叠比例 + 去重（语义去重） + 重排序 |
| 大知识库检索太慢 | 向量索引优化 + 缓存 + 分布式检索 + 预过滤 |
| 返回重复结果 | 去重（MinHash/LSH）+ 多样性重排序（MMR） |
| 需要按用户权限控制文档访问 | 元数据过滤 + 用户-文档映射 + 检索前鉴权 |
| 领域专业术语检索失败 | 领域微调 Embedding + 添加同义词/缩写映射 + 混合搜索 |
| 需要处理图片和表格 | 多模态 RAG + 表格解析 + OCR + 文档布局理解 |
| 知识库频繁更新需版本管理 | 文档版本化 + 增量索引 + 向量数据库 TTL + 变更检测 |
| 多跳推理失败 | 迭代检索 + 查询分解 + Graph RAG + Self-RAG |
| 多源文档返回矛盾答案 | 来源权重 + 置信度评估 + 矛盾检测 + 多答案呈现 |
| 知识库更新后仍返回旧答案 | 增量索引更新 + 版本标记 + 定期全量重建 |
| PDF 解析表格和布局失败 | 多解析器（Unstructured/PymuPDF）+ OCR + 表格专用解析 |
