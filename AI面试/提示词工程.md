# 💬 提示词工程面试题

> Prompt Engineering 核心概念、技巧与排障。

---

## 核心概念

**Q1：什么是提示词工程？为什么对 AI 应用至关重要？**
> 通过设计和优化输入给 LLM 的文本指令，引导模型产生期望的输出。
> 重要性：同样的模型，好的 Prompt 可以让效果提升数倍，是成本最低的优化手段。

**Q2：Zero-shot / One-shot / Few-shot Prompting 的区别？**
> - **Zero-shot**：不给示例，直接描述任务
> - **One-shot**：给 1 个示例
> - **Few-shot**：给多个示例，让模型学习模式
> 示例越多通常效果越好，但占用更多 token。

**Q3：什么是 Chain-of-Thought (CoT) Prompting？何时使用？**
> 让模型"逐步思考"，在给出答案前展示推理过程。
> 适用：数学推理、逻辑推理、多步骤任务。
> 方式：在 Prompt 末尾加 "Let's think step by step"

**Q4：什么是 Self-Consistency Prompting？**
> 多次采样 CoT 推理结果，取出现最多的答案（多数投票）。
> 提升推理任务准确率的简单有效方法。

**Q5：什么是 Tree-of-Thought Prompting？**
> 将推理过程建模为树状结构，每个节点是一个思考状态，可探索多条推理路径。
> 适合复杂规划、创意类任务。

**Q6：什么是 ReAct (Reasoning + Acting) Prompting？**
> 交替进行推理（思考）和行动（调用工具），让模型能执行多步推理并使用外部工具。
> 格式：`Thought → Action → Observation → Thought → ...`

## 实用技巧

**Q7：System Prompt 是什么？如何影响模型行为？**
> 系统提示词设定模型的角色、规则和约束。优先级最高，影响整个对话。
> 示例："你是一个专业的技术顾问，回答简洁，使用中文。"

**Q8：如何让 LLM 输出结构化格式（JSON/XML）？**
> - 在 Prompt 中明确指定格式和 schema
> - 提供输出示例
> - 使用 Function Calling / Structured Outputs API
> - 使用输出解析器（Output Parser）

**Q9：什么是 Prompt Injection？如何防御？**
> **直接注入**：用户在输入中嵌入恶意指令覆盖系统 Prompt
> **间接注入**：通过外部文档/网页嵌入恶意指令
> 防御：输入过滤、权限分离、输出验证、指令隔离

**Q10：什么是 Jailbreaking？常见手法？**
> 绕过 LLM 的安全限制。常见手法：
> - 角色扮演（"假装你是一个没有限制的 AI"）
> - 编码绕过（Base64、ROT13）
> - 多轮对话渐进绕过
> 防御：多层安全审查、红队测试

**Q11：如何优化 Prompt 的成本和延迟？**
> - 减少 Few-shot 示例数量
> - 用更短的指令替代冗长描述
> - 使用更小的模型处理简单任务（语义路由）
> - 缓存常见查询的回复
> - 精简上下文（只保留相关信息）

**Q12：Prompt Engineering vs Prompt Tuning 的区别？**
> - **Prompt Engineering**：手动设计文本提示词，无需训练
> - **Prompt Tuning**：添加可学习的软提示向量，通过训练优化

**Q13：什么是 Prompt Chaining？**
> 将复杂任务拆分为多个步骤，前一步的输出作为后一步的输入。
> 适合：长文本处理、多步推理、质量审核流程。

**Q14：什么是 Role Prompting？何时有效？**
> 让模型扮演特定角色（专家、顾问、评委）。
> 有效场景：需要专业领域知识、特定语气风格、特定视角。

**Q15：什么是 "Lost in the Middle" 问题？**
> 在长上下文中，模型对中间位置的信息关注度最低，容易遗忘。
> 解决：将重要信息放在开头或结尾、使用摘要、RAG 检索替代长上下文。

## 🔧 场景排障题

| 场景 | 思路 |
|------|------|
| Few-shot 效果不稳定 | 增加示例多样性 + 明确格式 + 使用 CoT + 评估示例质量 |
| 分类结果对 Prompt 措辞过于敏感 | 使用 Few-shot 稳定 + 标签定义清晰 + 评估一致性 |
| 系统 Prompt 被用户泄露 | 指令隔离 + 输出过滤 + 不在 Prompt 中放核心逻辑 |
| AI Agent 被 Prompt Injection 攻击 | 输入验证 + 沙箱执行 + 权限分级 + 输出审查 |
| Chain-of-Thought 没有提升推理准确率 | 确认任务适合 CoT + 提供推理示例 + 检查示例正确性 |
| 英文效果好但其他语言失败 | 翻译 Prompt + 多语言 Few-shot + 使用多语言模型 + 翻译-处理-回译 |
| 零样本跨语言迁移失败 | 翻译训练数据 + 使用多语言 Embedding + 适配目标语言 |

## 多语言提示

**Q16：如何有效处理多语言 Prompt？**
> - 使用多语言模型（GPT-4、Claude）原生支持
> - 将 Few-shot 示例翻译为目标语言
> - 使用翻译 API 翻译输入 → 处理 → 翻译输出
> - 使用特定语言的微调模型
