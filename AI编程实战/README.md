## 一、核心理念：Vibe Coding

### 什么是 Vibe Coding？

简单说，就是**"用说话来编程"**。

传统编程需要你掌握语法、框架、调试技巧；而现在，你只需要**用自然语言描述你想要什么**，AI 就能帮你生成代码。

> 大语言模型（LLM）的出现，让开发不再是"技术大神的专属"。曾经最难的是"怎么写代码"，现在最难的是"**你要做什么**"。

### 编程语言正在变成自然语言

以前做软件，门槛很高：你要懂编程、懂算法，还得有几年的项目经验。现在不一样了，只要你有想法，AI 就能帮你写代码。

### 谁适合学？

| 身份 | AI 编程能帮你 |
|------|-------------|
| 学生 | 作业、比赛、创业，自己动手做项目 |
| 职场人 | 重复工作自动化，效率提升，副业开发 |
| 产品经理/设计师 | 想法快速变 Demo，给老板/客户看 |
| 创业者/中小企业主 | 低成本验证想法，不花几万找外包做 MVP |
| 老师/教育工作者 | 教学工具、课件、自动化出题 |
| 医生/律师/专业工作者 | 专业流程自动化，效率工具 |
| 任何人 | 用 AI 解决生活/工作中的具体问题 |

### 🛠️ 第一阶段：做出你的第一个产品原型

**目标**：从零到做出一个"能跑、能给别人演示"的产品

**适合**：完全没编程基础，或只会一点点但不太自信

#### 1.1 学会 AI 编程工具

> 参考：[AI工具大全](../AI工具大全/README.md)

| 工具 | 特点 | 适合 |
|------|------|------|
| **Trae** | 国产免费，中文友好 | 初学者首选 |
| **Cursor** | AI 代码编辑器，最热门 | 有一定基础的开发者 |
| **Claude Code** | Anthropic 出品，代码能力强 | 高级开发者 |
| **GitHub Copilot** | VS Code 插件，代码补全 | 日常开发辅助 |
| **Windsurf** | 新兴 AI IDE | 全栈开发者 |
| **VS Code + AI 插件** | 原版 VS Code + AI 扩展 | 想保持原有习惯的开发者 |

**核心区别：普通 IDE vs AI IDE**

| 普通IDE | AI IDE |
|---------|--------|
| 报错自己查 | 把报错丢给 AI，让它分析原因 |
| 加功能自己写 | 说"做一个登录页"，AI 生成代码 |
| 改代码自己找文件 | AI 自动定位、跨文件修改 |
| 配置环境自己查文档 | AI 直接帮你配好 |

#### 1.2 找到好的产品想法

- 从身边痛点出发：你每天有什么事觉得烦？
- 参考已有产品：有没有什么工具你觉得不好用？
- 用户调研：问问身边人有什么需求
- MVP 思维：用最小成本验证想法

#### 1.3 搭建产品原型

- 从简单页面开始，逐步做出能交互的多页面应用
- 使用 AI 帮你生成 UI、处理交互逻辑
- 学会基本的 HTML/CSS/JavaScript 概念

#### 1.4 接入 AI 能力

- 文本生成（调用 ChatGPT/Claude API）
- 文生图（调用 DALL-E/Stable Diffusion）
- 智能对话（接入聊天机器人）
- 语音识别（接入 Whisper 等）

#### 1.5 完整项目实战

独立完成一个完整项目，从创意到可运行的原型。

## 三、提问的艺术

### 为什么提问是 AI 时代的必备技能？

> 同一份代码、同一个报错，**你怎么提问，几乎决定了 AI 能给出怎样的答案**。

- **现实很少有完整文档**：需求模糊、代码半成品、错误信息零散
- **AI 是随身导师 + 同事**：会提问 = 高质量结对编程
- **能力上限由沟通决定**：越能提供关键信息，答案越可用

### 常见误区

| ❌ 错误提问 | ✅ 正确提问 |
|-----------|-----------|
| "为啥报错？" | "运行以下代码后报了这个错误：[完整报错信息]，请帮我分析原因" |
| "帮我写个登录功能" | "我要用 React + Supabase 实现登录功能，支持邮箱和手机号，请生成代码" |
| "代码跑不起来" | "运行时报错：[截图+报错文本]，环境是 Node.js 18，请帮我解决" |

### 提示词技巧

**学习型提问：**
> "请先用 5 句话讲清楚这个概念，再给我出几个问题验证我理解对了没。"

**调试型提问：**
> "我把完整的报错信息贴在下面，请先分析可能的原因，再给出具体的修改方案。"

**架构型提问：**
> "我想做一个 [产品]，用户是 [人群]，核心功能是 [功能]，请帮我设计技术方案。"

### 截图 vs 复制粘贴

| 方式 | 适用场景 | 要求 |
|------|---------|------|
| **复制粘贴** | 报错堆栈、日志、代码、配置 | 尽量完整 |
| **截图** | UI 布局问题、交互异常 | 截全屏 + 标注重点 + 文字说明 |

> ⚠️ 截图需要 AI 支持多模态能力（Claude、GPT-4o、Gemini、通义千问等）

## 四、项目制学习：为什么重要？

> 真实工作中的许多任务，本质上就是在高度不确定的环境下解决从未见过的问题。

项目制学习的核心价值：

1. **锻炼拆解能力** - 模糊需求 → 可执行方案
2. **阅读代码能力** - 理解和改造中大型代码库
3. **完整闭环** - 从创意到上线的全流程体验
4. **真实模拟** - 提前体验"职场社会毒打"
